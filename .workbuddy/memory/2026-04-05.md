## auto-login JWE 修复 (commit 1417daa)

- **根因**：next-auth v5 (Auth.js) 使用 **JWE 加密**（A256CBC-HS512），不是普通 JWS 签名
- 之前的 auto-login 用 `jose.SignJWT`（JWS），`auth()` 解密时必然失败返回 null
- **修复**：改用 `jose.EncryptJWT` + `@panva/hkdf` 密钥派生，与 Auth.js 内部 `getDerivedEncryptionKey` 完全一致
- salt 使用 cookie name（`next-auth.session-token`），密钥长度 64 bytes
- 修复后 auto-login 签发的 token 能被 `auth()` 正确解密，session API 返回有效用户

## 支付后 Pro 状态不同步 Bug 修复 (commit 72cd19d)

### 根因：
支付成功后，verify API 正确更新了 DB（`subscription_tier=pro`），但 SiteNav 和 PremiumWall 从 `/api/auth/session` 读取的 tier 仍然是 `free`。原因是 NextAuth JWT cookie 中缓存了旧的 `subscriptionTier=free`，而 `/api/auth/session` 端点只解码现有 JWT 返回，不会因为 JWT callback 更新了内存中的 token 就重新签发 cookie。

### 修复方案：
1. **新建 `/api/user/tier` 端点** — 直接从 DB 读取 `subscription_tier`，绕过 JWT 缓存
2. **SiteNav.fetchSession** — 获取 session 后额外调用 `/api/user/tier`，用 DB 值覆盖 JWT 中的 tier
3. **PremiumWall** — 同样改为 session + tier API 双重查询
4. Dashboard Account 页面无需修改（已用 `/api/user/profile` 直接查 DB）
