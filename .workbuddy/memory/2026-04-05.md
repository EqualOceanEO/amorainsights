## Smart SubscribeBox 改造 (commit 6e10cba)

### 核心逻辑：
- **已登录用户**：显示邮箱 + 订阅状态 + 订阅/退订按钮 + Dashboard 链接，不再显示输入框
- **未登录用户**：正常输入框，提交时先调 `/api/auth/auto-login` 检查是否已注册
  - 已注册 → 自动创建 JWT session（jose 库，无需密码）+ 确保订阅 + 显示"Welcome back"
  - 新用户 → 走 `/api/subscribe` 正常流程

### 新增 API：
- `POST /api/auth/auto-login` — 通过邮箱查找用户，存在则签发 HS256 JWT + 设置 next-auth session cookie，不存在返回 `{exists: false}`
  - 使用 `jose` 库的 SignJWT（避免 next-auth jwt encode 的类型问题）

### SubscribeBox 变更：
- 组件 mount 时获取 `/api/auth/session` + `/api/user/profile`（获取 newsletter 状态）
- 支持 compact 和 card 两种布局的登录态展示
- 登录用户 card 布局：头像首字母 + 邮箱 + PRO 标签 + 订阅状态栏 + 操作按钮

## auto-login JWE 修复 (commit 1417daa)

- **根因**：next-auth v5 (Auth.js) 使用 **JWE 加密**（A256CBC-HS512），不是普通 JWS 签名
- 之前的 auto-login 用 `jose.SignJWT`（JWS），`auth()` 解密时必然失败返回 null
- **修复**：改用 `jose.EncryptJWT` + `@panva/hkdf` 密钥派生，与 Auth.js 内部 `getDerivedEncryptionKey` 完全一致
- salt 使用 cookie name（`next-auth.session-token`），密钥长度 64 bytes
- 修复后 auto-login 签发的 token 能被 `auth()` 正确解密，session API 返回有效用户
