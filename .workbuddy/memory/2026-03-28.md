## AMORA Report Template v1.0 创建（2026-03-28）

建立行业研究报告标准化模板，与企业评估框架 AMORA Score 并行：

**五维框架：**
- **A** - Advancement（技术先进性）：技术壁垒、路线分歧、卡脖子分析、**世界模型**
- **M** - Mapping（产业链生态位）：上下游分布、中美对比、空白机会
- **O** - Operations（商业化运营）：客户结构、ROI、落地案例
- **R** - Reach（市场容量）：规模预测、方法论、三情景
- **A** - Assets（资本价值）：财务对比、估值排名、资本效率

**核心原则：**
- 框架固化（五维字母+章节顺序）→ 建立品牌识别
- 内容灵活（子章节+数据呈现）→ 保持行业针对性
- 每章必须有"核心发现"段 + 中美对比结论 + So What

**v1.1 更新（2026-03-28）**：
- A章新增**世界模型（World Model）**评估维度
- 适用于具身智能、机器人、自动驾驶等物理世界交互行业
- 评估五维度：环境理解、物理预测、规划决策、仿真训练、数据引擎

## industries.ts L2 数据修复（2026-03-28）

- Manufacturing `level2` 原来是 `[]`，补全为数据库实际值：Industrial Robots / Additive Manufacturing / Digital Twin / IIoT & Smart Factory / Autonomous Vehicles
- Advanced Materials L2 名称与 DB 不一致：`Carbon Fiber & Composites` → `Carbon Fiber`，`Graphene & Carbon Nanocomposites` → `Graphene`
- **规则**：`industries.ts` 里的 L2 名称必须与 `companies.sub_sector` 字段值完全一致，否则过滤不到数据

**根因**：Vercel 环境变量 `MIGRATION_SECRET` 未配置，导致 cron 调用 `/api/admin/news-generator?secret=run-migration-now` 始终返回 401，从 3月26日起无新数据。

**临时修复**：用 Bearer service_role_key 方式手动触发一次，成功生成 20 条新闻（6行业+8公司+6插入）。

**根治方案（待操作）**：在 Vercel Dashboard → Settings → Environment Variables 添加 `MIGRATION_SECRET=run-migration-now`，然后 Redeploy。

**触发脚本逻辑**：Bearer token = SUPABASE_SERVICE_ROLE_KEY，鉴权走 `tokenMatches` 分支。
