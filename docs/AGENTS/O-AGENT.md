## 一、核心职责

**Part O 是报告的"商业落地层"**，回答"谁能真正赚钱，谁只是有收入"。负责：

1. **商业模式分析** — 各玩家的商业路径，哪种模式最有效
2. **收入结构拆解** — 硬件 vs 软件 vs 服务，真实收入占比
3. **客户结构洞察** — 客户是谁（科研/企业/政府/消费者），哪种客户最有价值
4. **单位经济模型（Unit Economics）** — 客户获取成本 vs 生命周期价值 vs ROI
5. **四大应用场景深度分析** — 最快商业化的场景 + 壁垒最高的场景
6. **量产障碍** — 从1台到100台，从100台到10000台的瓶颈在哪
7. **中国 vs 北美商业化对比** — 两国市场商业化路径的根本差异

## 二、内容标准

### 必含模块（按顺序）

```
## Part O：商业化运营分析

### O-1. 商业模式图谱
  - 本行业主流商业模式（至少3种）
  - 每种模式：定义 + 代表公司 + 优劣势 + 可持续性判断
  - 商业模式演进趋势
  - 表格：商业模式 × 公司矩阵

### O-2. 收入结构深度拆解
  - 硬件收入 vs 软件/SaaS 收入 vs 服务收入
  - 真实数据：各收入类别占比（用财报/融资数据）
  - 哪类收入利润率最高？
  - 收入的可重复性（Recurring vs One-time）

### O-3. 客户结构洞察
  - 客户分类：科研教育 / 企业应用 / 政府 / 消费者
  - 各客户类型：需求量级 + 决策周期 + 付费意愿
  - 谁是最快付费的客户？（以人形机器人为例：科研教育最快，企业导览其次，工业替代目前极少）
  - 客户集中度风险（是否有单一大客户依赖）

### O-4. 单位经济模型（Unit Economics）
  - 客户获取成本（CAC）：各公司公开数据推算
  - 客户生命周期价值（LTV）：估算方式 + 数据来源
  - LTV/CAC 比值（行业健康度指标）
  - 盈亏平衡时间线：从首台部署到单台盈利需要多久？

### O-5. 应用场景商业化分析（四大场景）
  每类场景：
  - 当前市场规模（已有真实收入）
  - 增长速度（YoY）
  - 主要玩家 + 代表订单
  - 进入壁垒（为什么难被复制）
  - 哪个场景最早商业化成熟？

### O-6. 量产障碍与规模化瓶颈
  - 从原型 → 小批量（< 100台）的核心障碍
  - 从批量 → 规模化（> 1,000台）的核心障碍
  - 供应链制约因素
  - 服务/维护网络建设的难度
  - 谁最有可能先突破量产瓶颈？

### O-7. 中国 vs 北美商业化路径对比
  - 中国市场：商业化速度 vs 市场深度
  - 北美市场：付费意愿 vs 监管环境
  - 跨国扩张的挑战
  - 哪个市场先跑通商业模式？
```

## 三、商业数据可信度标准

### 财务数据获取优先级
| 优先级 | 来源 | 可靠性 |
|-------|------|-------|
| ★★★★★ | 上市公司财报（10-K/10-Q） | 高，直接引用 |
| ★★★★ | 上市公司 IPO 招股书 | 高，详细信息 |
| ★★★★ | 上市公司投资者日/业绩会 | 高，可引用 |
| ★★★ | 经审计融资公告（金额+轮次） | 中，金额可能夸大 |
| ★★ | 未审计新闻稿数据 | 低，需交叉验证 |
| ★ | 企业自报（官网/新闻） | 低，不可单独使用 |

### 估算方法规范
**当无法获取真实数据时：**
```
"我们估算 {公司} 的 {指标} 约为 {估算值}，
基于：{推算方法}，来源：{参考数据}。
此为估算值，实际数字可能存在 ±30% 的偏差。"
```
