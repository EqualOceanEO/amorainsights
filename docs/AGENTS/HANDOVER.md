# 智能体交接协议 — HANDOVER.md

> **目的：** 当新的智能体接替当前智能体时，确保上下文不丢失、无缝衔接
> **适用范围：** Agent 版本升级、跨团队交接、生产事故恢复
> **版本：** v1.0 | **日期：** 2026-04-12

---

## 一、交接触发条件

以下情况必须执行交接协议：

| 触发条件 | 说明 |
|---------|------|
| 版本升级 | Agent 能力包更新，需同步新版本 |
| 连续失败 | 同一任务连续失败 3 次 |
| 超时未完成 | 任务超过 SLA 仍未交付 |
| 人工干预 | 人工介入修改了 Agent 输出 |
| 事故恢复 | 系统崩溃后恢复，需重建上下文 |
| 人员变更 | 新智能体接替现有智能体 |

## 三、接收 Agent 操作规范

### 第一步：读取交接文档
```
读取最新的 HANDOVER.md（位于 docs/AGENTS/ 目录）
```

### 第二步：验证上下文
```
□ 确认报告名称、行业、数据截止日期正确
□ 确认章节顺序（Part M → A → O → R → A2）
□ 确认 AMORA Score 评分进度
□ 确认质量核验状态
```

### 第三步：快速自检
```
□ 读取 Agent 专属 brain 文档（docs/AGENTS/{X}-AGENT.md）
□ 对照质量检查清单快速验证上一 Agent 输出
□ 如有问题，标记并上报 Orchestrator
```

### 第四步：继续生产
```
□ 从交接点继续执行，不重复已完成工作
□ 更新交接文档（注明"已完成交接"）
□ 继续执行质量核验流程
```

## 四、常见交接问题处理

| 场景 | 处理方式 |
|------|---------|
| 交接文档缺失 | 立即联系 Orchestrator，从 Supabase 重建上下文 |
| 数据来源不明 | 降级为"待核实"，在输出中标注，待后续核实 |
| 评分缺失 | 按 Agent 规范自行评估，不留空白 |
| 跨语言版本不一致 | 优先对齐英文版，中文版参照英文版重新翻译 |
| 版本冲突 | 以最新版本为准，旧版本归档至 `archive/` 目录 |
