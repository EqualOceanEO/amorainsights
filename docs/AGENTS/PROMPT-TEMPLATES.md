# AMORA 各章节标准提示词模板

> **用途：** 每次启动新报告时，直接复制对应章节模板，填入行业参数即可开始写作
> **版本：** v1.0 | **日期：** 2026-04-12

## 一、Part M — Mapping 标准提示词

```markdown
# 任务：撰写 Part M — 产业链生态位分析

## 行业信息
- 行业名称：{行业英文名} / {行业中文名}
- 报告标题：The {行业英文名} Innovation Index {年份}
- 核心问题：{一句话描述这份报告要回答的关键问题}
- 覆盖公司：{Top 10 公司清单，格式：序号. 公司名（国家）— 核心产品/定位}

## 数据截止
- {YYYY-MM-DD}

## 参考资源
{资源清单，格式：- 资源名（类型）：获取方式}

## 内容要求
请严格按照 docs/AGENTS/M-AGENT.md 中的标准撰写，包含：
1. M-1 行业定义与边界
2. M-2 产业链全景图（包含 ASCII/Mermaid 结构图）
3. M-3 生态位分布矩阵
4. M-4 主要玩家深度档案（Top 10，每家150-250字）
5. M-5 供应链脆弱性分析
6. M-6 中美生态位对比
7. M-7 Part M 付费钩子（冲击力数字 + 一句话结论）

## 语言要求
- 中文版本：{CN}，正文零英文字符（CSS属性除外）
- 英文版本：{EN}，正文零中文字符
- 参考资源中的机构名保留英文

## 数据来源要求
- 所有数字必须附来源（来源类型优先级：★★★★★政府数据 > ★★★★财报 > ★★★行业报告）
- 禁止使用：PitchBook / CB Insights / Gartner / McKinsey 等未授权数据
- 缺数据项标注"数据待补充"，不得捏造

## 输出格式
文件命名：HRI-{年份}-{行业缩写}-PartM-{语言}.md
头部元数据：part / chapter / industry / version / date / author / word_count / data_cutoff / status

## 字数要求
- 中文版 ≥ 4,000 字（不含公司档案表格）
- 英文版 ≥ 4,000 词（不含公司档案表格）

## 特别注意
- M-7 付费钩子是整份报告的转化关键，要精准有力
- 产业链图谱要包含所有关键环节，无重大遗漏
- 中美对比要有具体的交叉点，不是泛泛而谈
```

## 二、Part A — Advancement 标准提示词

```markdown
# 任务：撰写 Part A — 技术先进性分析

## 行业信息
- 行业名称：{行业英文名}
- 核心问题：{一句话描述这份报告要回答的技术关键问题}

## 技术维度（如有特定技术重点）
- 核心技术清单：{技术1} / {技术2} / {技术3}
- 世界模型/端到端是否适用：{是/否}

## 参考资源
{资源清单，重点包括：}
- USPTO 专利数据库检索：公司名 + 专利数量 + 引用次数
- arXiv / 顶级会议论文：关键词检索链接
- 技术白皮书：来源

## 内容要求
严格按照 docs/AGENTS/A-AGENT.md 标准，包含：
1. A-1 核心技术图谱（含成熟度曲线）
2. A-2 技术路线之争（主流 vs 颠覆性路径）
3. A-3 技术壁垒评估（专利/数据/人才/生态四维度）
4. A-4 中美技术差距真实评估（数据驱动，不是叙事）
5. A-5 世界模型/端到端进展（如适用）
6. A-6 关键技术事件时间线（过去12个月，5-10项）
7. A-7 技术领先者 vs 模仿者

## 语言要求
- 中文版：技术术语首次出现附英文（例：Transformer（变换器））
- 英文版：使用标准技术术语
- 正文零跨语言字符混入

## 数据要求
- 专利分析需真实 USPTO 数据（非估算）
- 技术成熟度判断需有依据
- 中美差距必须有具体数据对比

## 输出格式
文件命名：HRI-{年份}-{行业缩写}-PartA-{语言}.md
字数：中文 ≥ 4,500字 / 英文 ≥ 4,500词
```

```markdown
## 商业模式重点
- 主要商业模式：{至少3种}
- 四大应用场景：{场景1} / {场景2} / {场景3} / {场景4}

## 参考资源
{资源清单，重点包括：}
- 上市公司财报（SEC EDGAR / 港交所 / A股）
- IPO 招股书
- 融资公告清单

## 内容要求
严格按照 docs/AGENTS/O-AGENT.md 标准，包含：
1. O-1 商业模式图谱（含商业模式 × 公司矩阵）
2. O-2 收入结构深度拆解（硬件 vs 软件 vs 服务）
3. O-3 客户结构洞察（科研/企业/政府/消费者）
4. O-4 单位经济模型（Unit Economics：CAC / LTV / ROI）
5. O-5 应用场景商业化分析（四大场景并列）
6. O-6 量产障碍与规模化瓶颈
7. O-7 中国 vs 北美商业化路径对比

## 数据要求
- 财务数据：上市公司优先，估算需说明方法
- 客户结构：揭示真相，不做美化（如实告知哪类客户最多）
- 单位经济模型：必须标注推算方法

## 输出格式
文件命名：HRI-{年份}-{行业缩写}-PartO-{语言}.md
字数：中文 ≥ 4,500字 / 英文 ≥ 4,500词
```

## 五、Part A2 — Assets + AMORA Score 标准提示词

```markdown
# 任务：撰写 Part A2 — 资本价值与 AMORA Score 评分

## 行业信息
- 行业名称：{行业英文名}
- 覆盖公司：Top 12 第一梯队公司（从 Part M 获取）
- 行业均分：待 A2-Agent 根据评分数据计算

## AMORA Score 评分任务

### 五维度评分（每家公司）
| 公司 | A（技术） | M（人才） | O（运营） | R（全球） | A（持续） |
|------|:-------:|:-------:|:-------:|:-------:|:-------:|
| 公司A | ? | ? | ? | ? | ? |
| 公司B | ... | ... | ... | ... | ... |

### 评分标准（详见 docs/AGENTS/A2-AGENT.md）
- 10分 = 全球前3，无明显短板
- 8分 = 行业领先，有若干短板
- 6分 = 行业平均，有明显短板
- 4分 = 落后于行业平均
- 1-2分 = 基本出局
- 缺数据 → 标注"数据待补充"，不得硬评

### 评分卡格式
必须包含：综合分 = (A+M+O+R+A) / 5，附行业均值，标出同环比变化

## 内容要求
严格按照 docs/AGENTS/A2-AGENT.md 标准，包含：
1. A2-1 主要玩家财务对比（Top 12，6维度）
2. A2-2 资本效率矩阵
3. A2-3 融资动态追踪（近12个月）
4. A2-4 估值方法论与争议
5. A2-5 AMORA Score 企业评分卡（**核心输出**）
6. A2-6 投资建议（附免责声明）
7. A2-7 附录：评分方法论（透明披露）

## 免责声明（必须放在 A2-6 最前面）
"以下内容仅供信息参考，不构成投资建议。AMORA 与提及公司均无利益关联。评分基于公开数据，不保证完整性。投资者应独立判断，市场有风险。"

## 数据要求
- 财务数据：上市公司财报优先
- 估值：上市公司用市值，非上市公司用最新融资估值
- 禁止 PitchBook / CB Insights 数据

## 输出格式
文件命名：HRI-{年份}-{行业缩写}-PartA2-{语言}.md
字数：中文 ≥ 5,200字 / 英文 ≥ 5,200词
```

## 六、通用写作规范（所有章节通用）

### 格式规范
```
- 标题层级：# 一级 → ## 二级 → ### 三级
- 表格：Markdown 标准表格
- 图表说明：图 X. {标题}  来源：{来源}
- 引用格式：（{公司}，{年份}）
- 参考资料列表：随章节附在末尾
```

### 禁止出现的内容
```
❌ PitchBook / CB Insights / Gartner / Forrester / McKinsey 数据
❌ 未经核实的市场数据
❌ 暗示投资建议的语言（"建议买入"、"目标价"等）
❌ 中英文混杂（正文层面）
❌ 泛泛而谈无数据的结论
❌ "行业普遍认为"类无法溯源的引用
```

### 数字与单位
```
- 金额：统一用美元（$），大数字用缩写（$1.2B / $50M）
- 百分比：%（附基准数字）
- CAGR：注明起止年份
- 日期：YYYY-MM-DD
```
