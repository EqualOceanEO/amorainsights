## 一、核心职责

**Part M 是整份报告的"入口层"**，是用户决定是否付费的关键钩子。负责：

1. **产业链全景图** — 谁在做，做什么，之间的关系是什么
2. **生态位分析** — 各玩家的定位、规模、优势对比
3. **供应链脆弱性识别** — 卡脖子环节在哪里
4. **中美生态位对比** — 两国玩家的差异化路径
5. **埋设付费钩子** — Part M 末尾必须有一个冲击力数字，勾起付费欲望

## 二、内容标准

### 必含模块（按顺序）

```
## Part M：产业链生态位分析

### M-1. 行业定义与边界
  - 行业是什么，不是什么（精确边界）
  - 核心价值链：上游 → 中游 → 下游
  - 本报告的覆盖范围（公司清单 + 地理范围）

### M-2. 产业链全景图
  - 产业链结构图（ASCII / Mermaid / SVG）
  - 各环节主要玩家（公司名 + 一句话定位）
  - 环节之间的依赖关系
  - 平台层 vs 应用层 vs 基础设施层

### M-3. 生态位分布矩阵
  - X轴 = 商业化程度（原型 → 小批量 → 大规模）
  - Y轴 = 技术深度（集成 → 自主研发 → 核心创新）
  - 气泡 = 营收规模 / 估值
  - 标注关键拐点事件

### M-4. 主要玩家深度档案（Top 10）
  每家公司：
  - 公司名（中英文）
  - 总部所在
  - 核心产品
  - 关键技术壁垒（1句话）
  - 最新融资/营收数据
  - 生态位标签（技术型 / 商业型 / 平台型）

### M-5. 供应链脆弱性分析
  - 上游核心零部件依赖（卡脖子清单）
  - 产能瓶颈在哪一环节
  - 地域集中风险
  - 国产替代进展

### M-6. 中美生态位对比
  - 中国玩家：优势领域 + 代表公司
  - 美国玩家：优势领域 + 代表公司
  - 两国差异化的根本原因
  - 交叉竞争点（中美正面交锋的细分市场）

### M-7. Part M 付费钩子（必须）
  在 Part M 末尾，单独一个区块：
  - 一个冲击力数字（如："仅 3/25 家追踪企业在真实场景部署 ≥50 台"）
  - 一句话结论（如："真正拉开差距的不是技术参数，是数据闭环速度"）
  - 不要给出答案——只给出问题，让读者想看 Part A
```

### 篇幅标准

## 三、数据来源规范

### 优先级（强制排序）

| 优先级 | 来源类型 | 举例 |
|-------|---------|------|
| ★★★★★ | 上市公司年报/10-K | SEC EDGAR 原始文件 |
| ★★★★ | 政府/监管公开数据 | USPTO 专利库、BLS、行业统计局 |
| ★★★ | 行业协会报告 | 注明版权，须有授权或为公开版本 |
| ★★ | 经同行评审学术论文 | 需注明 DOI |
| ★ | 企业官网/新闻稿 | 需交叉验证，不可单独作为核心数据 |

### 禁止使用
- ❌ PitchBook / CB Insights（未授权商业数据库）
- ❌ Gartner / Forrester / McKinsey（需商业授权）
- ❌ 未核实的社交媒体数据
- ❌ 竞争对手内部信息
