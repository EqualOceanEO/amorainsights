## 一、核心职责

**Part A 是报告的"技术深度层"**，回答"这个行业的核心技术是什么，谁在真正领先"。负责：

1. **核心技术识别** — 本行业最重要的3-5项技术及其成熟度
2. **技术路线图** — 主流技术路径 vs 颠覆性路径
3. **技术差距分析** — 中美技术差距的真实大小（用数据说话，不是叙事）
4. **技术壁垒评估** — 哪些是真正的壁垒，哪些是护城河
5. **世界模型/端到端进展** — 是否在从"专用"走向"通用"（如人形机器人）
6. **关键技术新闻时间线** — 过去12个月最重要的技术事件

## 二、内容标准

### 必含模块（按顺序）

```
## Part A：技术先进性分析

### A-1. 核心技术图谱
  - 本行业核心技术清单（3-5项）
  - 每项技术：定义 + 成熟度（实验室/原型/商业化/规模化）
  - 技术成熟度曲线（Visual）
  - 各项技术之间的关系（谁依赖谁）

### A-2. 技术路线之争
  - 主流技术路径（大多数玩家选择）
  - 颠覆性/替代技术路径（小众但潜力）
  - 路线之争的胜负关键（什么变量决定哪条路胜出）
  - 关键技术决策节点（未来12-18个月）

### A-3. 技术壁垒评估
  - 专利壁垒（专利数量、质量、覆盖范围）
  - 数据壁垒（真实场景数据的质量和数量）
  - 人才壁垒（顶尖人才集中度）
  - 生态壁垒（开发者生态、合作伙伴）
  - 每项壁垒：壁垒强度（高/中/低）+ 主要持有者

### A-4. 中美技术差距真实评估
  - 各技术维度中美对比表格
  - 中国技术短板在哪里（具体，不是泛泛）
  - 美国技术短板在哪里
  - 差距在扩大还是在收窄？（用数据回答）

### A-5. 世界模型 / 端到端进展（如适用）
  - 本行业是否在从"专用"走向"通用"？
  - 世界模型进展到了什么程度？
  - 数据闭环的挑战（仿真 vs 真实世界数据）
  - 下一个技术突破点预测（具体，不模糊）

### A-6. 关键技术事件时间线
  - 过去12个月最重要的5-10项技术进展
  - 每项：时间 + 事件 + 意义 + 来源
  - 时间线可视化

### A-7. 技术领先者 vs 模仿者
  - 真正做原创技术的公司（3-5家）
  - 做工程化/规模化落地的公司（3-5家）
  - 两者的根本区别是什么
```

## 三、技术可信度评估框架

每个技术声明必须回答：

```
1. 来源是什么？（★★★★★ 政府数据 > ★★★★ 财报 > ★★★ 行业报告）
2. 时间新鲜吗？（超过18个月的数据需标注时效性）
3. 是否被质疑？（有反例吗？）
4. 能被独立核实吗？（专利号/论文DOI/官方发布）

结论声明格式：
"截至 {日期}，{公司/机构} 的 {指标} 为 {数据}，来源：{来源}"
```

## 四、数据来源规范

### 优先顺序
| 优先级 | 来源类型 |
|-------|---------|
| ★★★★★ | USPTO 专利数据库（可按公司检索） |
| ★★★★★ | arXiv / 顶级会议论文（CVPR, NeurIPS, ICRA 等） |
| ★★★★ | 上市公司技术白皮书 / 官方技术发布 |
| ★★★ | 行业技术媒体（TechCrunch, IEEE Spectrum, The Verge） |
| ★★ | 行业分析师报告（注明来源性质） |

### 专利分析方法
- 用 USPTO 检索目标公司的专利数量、质量（引用次数）、覆盖领域
- 对比中美 Top 5 公司的专利布局差异
- 标注关键专利家族（被引用次数最多的专利）
